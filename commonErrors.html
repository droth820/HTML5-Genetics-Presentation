Programming Errors

3 types of programming errors:
  • Syntax errors = bad grammar
    • Did you forget a semicolon?
    • File name != class name
    • Misspelled words
    • Forgot to capitalize something
    • Missing key words
    • Compiler will find them
    • Must fix before program will run

Programming Errors Activity

Error Checking Algorithm


Your list should include the following:
  • File name matches class name
  • All code is spelled correctly
  • All code is capitalized correctly
  • All statements end in a semicolon
  • Keywords are included
  • Strings are enclosed in "quotation marks"
  • No extra punctuation marks
  • All header open-braces are paired with closed-braces

Comments

Documenting your program's code helps you to remember why you wrote lines of code the way you did.
  • nonexecuting statements added for documentation
  • Leave notes for yourself and others
  • At the very least should include:
    • Author's name
    • Date created
    • Class name
    • Purpose of each method within the class
  • Comment out (Compiler won't translate)
  • Line comments = //
  • Block comments = /* Comment here */
